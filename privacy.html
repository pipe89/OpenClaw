OpenClaw Privacy Policy

This Privacy Policy explains how OpenClaw handles user information.

What OpenClaw accesses

OpenClaw accesses Google account data only to the extent necessary to provide the functionality requested by the user through the command-line application.

Authentication data

OpenClaw stores OAuth tokens locally on the user's device so the user can remain authenticated between sessions. These tokens are used only to access Google APIs on behalf of the user, based on the permissions the user has granted.

How data is used

Data accessed through Google APIs is used only to provide the features of OpenClaw that the user has requested. OpenClaw does not sell user data.

How data is shared

OpenClaw does not share Google user data with third parties except when required to operate the application as requested by the user or when required by law.

Data storage

Authentication tokens and related local configuration may be stored on the user's machine. The user is responsible for securing their local device and files.

Revoking access

A user can revoke OpenClaw's access at any time through their Google account permissions settings. After access is revoked, previously stored tokens may stop working and the user may need to sign in again to continue using the application.

Contact

If you have questions about this Privacy Policy, contact the application owner through the project repository or the contact details provided in the Google OAuth consent screen.
